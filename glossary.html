專有名詞詞彙表 (Glossary)

本頁面旨在解釋 API 安全領域中常見的專有名詞與縮寫，幫助您更好地理解本專案的內容。

API (Application Programming Interface) - 應用程式介面

一種允許不同軟體應用程式之間互相溝通、交換資料的規則與工具集合。在現代架構中，API 是串連微服務、行動應用和第三方服務的關鍵橋樑。

OWASP (Open Web Application Security Project) - 開放式 Web 應用程式安全專案

一個全球性的非營利組織，致力於改善軟體安全。其發布的 OWASP Top 10 (包括針對 Web 應用和 API 的版本) 是業界公認的最關鍵安全風險列表。

BOLA (Broken Object Level Authorization) - 權限不當的物件級授權 (OWASP API #1)

一種安全漏洞，當 API 端點允許使用者存取他們不應有權限存取的特定資料物件時發生。例如，使用者 A 透過修改 API 請求中的 ID，成功存取了使用者 B 的資料。

MFA (Multi-Factor Authentication) - 多因素驗證

一種安全機制，要求使用者在登入時提供兩種或兩種以上的驗證因素 (例如：密碼 + 手機驗證碼)，以增強帳戶的安全性。

NIST CSF (National Institute of Standards and Technology Cybersecurity Framework) - 美國國家標準與技術研究院網路安全框架

一個由美國政府機構開發的、旨在幫助組織管理和降低網路安全風險的框架。它提供了一套標準、指南和最佳實踐，分為識別、保護、偵測、應變、復原五大功能。

影子 API (Shadow API)

指那些未被 IT 或安全團隊正式記錄、管理和監控的 API。它們可能是由開發人員為了測試或臨時需求而建立，但從未被移除，因此成為了無人看管的安全盲點。

殭屍 API (Zombie API)

指那些已經過時、被新版本取代，但仍未被停用的舊版 API。因為它們通常不再被積極維護，所以很容易被駭客發現並利用其中的已知漏洞。
